## Comment exécuter

### Tout-en-un (recommandé)

```bash
# Lance tout : build, serveur, setup, Pa11y, Lighthouse, rename, teardown
pnpm a11y

# Pa11y seulement
pnpm a11y:pa11y-only

# Lighthouse seulement
pnpm a11y:lighthouse-only
```

### Analyse des résultats

```bash
# Rapport complet (scores + Core Web Vitals par page)
pnpm a11y:lighthouse:report

# Idem + détails des échecs de contraste couleur
pnpm a11y:lighthouse:report:contrast

# Générer le rapport texte complet dans tests/reports/lighthouse-report.txt
pnpm a11y:report
```

Le rapport texte (`lighthouse-report.txt`) inclut :
- Tableau des scores par catégorie et par page
- Core Web Vitals (FCP, LCP, SI, TBT, CLS) avec indicateurs 🟢🟡🔴
- Résumé avec moyennes
- Détail des audits échoués par page (score < 90)
- Détail des problèmes d’accessibilité

L'orchestrateur (`tests/a11y/run.cjs`) :
1. Détecte si un serveur tourne déjà sur le port 4321
2. Si non → build + démarre le serveur en arrière-plan
3. Seed les users de test + exporte les cookies
4. Exécute les audits demandés
5. Renomme les rapports LHCI
6. Teardown (supprime les users)
7. Arrête le serveur (seulement s'il l'a démarré)

### Commandes individuelles

```bash
# Prérequis : serveur en cours sur localhost:4321
# (pnpm build && pnpm preview)

# 1. Seed users + export cookies
pnpm a11y:setup

# 2. Pa11y-ci (WCAG AAA, 40 URLs)
pnpm a11y:pa11y

# 3. Lighthouse CI — pages publiques (26 URLs)
pnpm a11y:lighthouse

# 4. Renommer les rapports
pnpm a11y:lighthouse:rename

# 5. Lighthouse CI — pages authentifiées + admin (12 URLs)
pnpm a11y:lighthouse:authed

# 6. Renommer les rapports authentifiés
pnpm a11y:lighthouse:rename

# 7. Cleanup
pnpm a11y:teardown

# 8. Analyser les rapports
pnpm a11y:lighthouse:report
pnpm a11y:lighthouse:report:contrast
```

## Pa11y-ci — Configuration

**Fichier** : `.pa11yci.cjs`

| Paramètre | Valeur | Rôle |
| :-- | :-- | :-- |
| `standard` | WCAG2AAA | Standard le plus strict (ratio contraste 7:1) |
| `runners` | `['axe']` | Utilise axe-core (plus fiable que htmlcs par défaut) |
| `timeout` | 30 000 ms | Timeout par page |
| `wait` | 1 000 ms | Attente après chargement pour JS asynchrone |
| `chromeLaunchConfig.executablePath` | Auto-détecté | Utilise le Chromium installé par Playwright |
| `chromeLaunchConfig.args` | `--no-sandbox --disable-setuid-sandbox` | Requis en CI (conteneur Docker) |

### Authentification Pa11y

Pa11y supporte les headers par URL. La config génère automatiquement :
- URLs publiques : aucun header
- URLs authentifiées : `headers: { Cookie: userCookie }`
- URLs admin : `headers: { Cookie: adminCookie }`

Les cookies sont lus depuis `.a11y-cookies.json` (créé par `pnpm a11y:setup`).

## Lighthouse CI — Configuration

**Fichier** : `lighthouserc.cjs` (public) + `tests/a11y/lhci-authed.cjs` (authentifié)

### Gates (seuils d'erreur)

```javascript
assertions: {
  'categories:performance':    ['error', { minScore: 0.9 }],
  'categories:accessibility':  ['error', { minScore: 0.9 }],
  'categories:best-practices': ['error', { minScore: 0.9 }],
  'categories:seo':            ['error', { minScore: 0.9 }],
}
```

### 3 batches séparés

| Batch | Script | URLs | Cookie | Config |
| :-- | :-- | --: | :-- | :-- |
| Public | `pnpm a11y:lighthouse` | 26 | Aucun | `lighthouserc.cjs` |
| Authenticated | `lhci-authed.cjs` | 8 | `userCookie` | Fichier temporaire `.lighthouseci-authed.json` |
| Admin | `lhci-authed.cjs` | 4 | `adminCookie` | Fichier temporaire `.lighthouseci-admin.json` |

> LHCI ne supporte pas les headers par URL dans la config. Le script `lhci-authed.cjs` génère un config JSON temporaire avec `extraHeaders: { Cookie: … }` pour chaque batch, puis le supprime après exécution.

### Détection Chrome

La fonction `findChrome()` dans `.pa11yci.cjs` et `lighthouserc.cjs` :
1. Vérifie `process.env.CHROMIUM_PATH`
2. Utilise l’API directe `require('@playwright/test').chromium.executablePath()` pour détecter le Chromium installé par Playwright
3. Fallback : laisse Puppeteer/Lighthouse utiliser Chrome système

### Rapports

- **Upload** : `temporary-public-storage` (lien public dans les logs CI)
- **Local** : `.lighthouseci/` (gitignored)
- **Nommage** : les fichiers `lhr-{timestamp}.html` sont renommés par `lhci-rename.cjs` :
  - `fr--home.html` (homepage française)
  - `en--auth--sign-in.html` (page connexion anglaise)
  - `ar--من-نحن.html` (page à-propos arabe)
  - `ar--auth--تسجيل-الدخول.html` (page connexion arabe)

## Setup — Seed Users

**Fichier** : `tests/a11y/setup.ts`

| User | Email | Rôle | Utilisé par |
| :-- | :-- | :-- | :-- |
| `SEED_USER` | `a11y-seed@test.com` | `user` | Pa11y (dashboard, profile), LHCI (authed batch) |
| `SEED_ADMIN` | `a11y-admin@test.com` | `admin` | Pa11y (admin), LHCI (admin batch) |

### Processus

1. Supprime les users précédents (idempotent)
2. Inscrit via `auth.api.signUpEmail()`
3. Force `emailVerified: true` en DB
4. Se connecte via HTTP `POST /api/auth/sign-in/email` (avec header `Origin` pour CSRF)
5. Extrait le cookie `Set-Cookie` de la réponse
6. Promeut l'admin via SQL `UPDATE user SET role = 'admin'`
7. Re-authentifie l'admin pour obtenir un cookie avec le rôle mis à jour
8. Écrit `{ userCookie, adminCookie }` dans `.a11y-cookies.json`

### Teardown

```bash
pnpm a11y:teardown     # = tsx tests/a11y/setup.ts --teardown
```

Supprime les lignes `account`, `session`, `user` pour les 2 emails seed + supprime `.a11y-cookies.json`.

### Corrections a11y appliquées

| Problème | Composant | Correctif |
| :-- | :-- | :-- |
| `aria-prohibited-attr` | `SideCarouselCards.astro` | Ajout `role="img"` sur le conteneur des étoiles |
| `image-redundant-alt` | `Brand.astro` | `alt=""` sur le logo (parent `<a>` a déjà le texte) |
| `target-size` | `HeroSection.astro` | Dots du carousel agrandis de 8px à 24px |
| `color-contrast` (profil) | `ProfilePage.astro` | Suppression de l'opacité `/70` sur `text-muted-foreground` |
| `heading-order` (profil) | `ProfilePage.astro` | `<h3>` → `<h2>` (export data + delete account) |
| `label` (profil) | `ProfilePage.astro` | Ajout `for="profile-email"` + `id="profile-email"` sur l'input email |
| `color-contrast` (auth/about) | Auth pages + About sections | Nouveau token `text-primary-deep` (dark gold ≥5:1 sur blanc) |
| CLS (legal) | `Brand.astro` | Logo `w-auto` → `w-[120px]` pour réserver l'espace |
